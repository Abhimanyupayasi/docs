## Get Started with Test Case Creation

Follow these steps to understand the complete flow — from writing your first test case to running it successfully.

---

### Step 1: Understand the Testing Flow

<AccordionGroup>
  <Accordion icon="workflow" title="1. Go to the NexGen QE tab">
    Open the Piedap platform and navigate to the **NexGen QE** from your project dashboard.  
    <img src="/imgs/quickstart/Nexgen.png" alt="Navigate to NexGen QE tab" />
  </Accordion>

  <Accordion icon="layout-dashboard" title="2. Go to the Design tab">
    Once inside QE, select the **Design** tab.  
    This is where all your features and functionalities are listed.  
    <img src="/imgs/quickstart/Nexgen.png" alt="Open the Design tab" />
  </Accordion>

  <Accordion icon="list-checks" title="3. Choose your feature">
    Pick the **feature** you want to create a test case for.  
    Clicking on it will redirect you to the **Test Case section** under the Design tab.  
    <img src="/imgs/quickstart/Nexgen.png" alt="Select a feature for test case" />
  </Accordion>

  <Accordion icon="file-pen" title="4. Write your first test case">
    Choose the **functionality** you want to test.  
    Click on **New** to create your first test case.  

    <Tip>Start with a simple web testing flow to understand the process clearly.</Tip>
    <img src="/imgs/quickstart/Nexgen.png" alt="Write a new test case" />
  </Accordion>

  <Accordion icon="window" title="5. Create the Test Case (Web Testing)">
    Begin with a **Web Testing** case.  
    Define the **test name**, **steps**, and **expected results**.  
    <img src="/imgs/quickstart/Nexgen.png" alt="Define web test case details" />
  </Accordion>

  <Accordion icon="arrow-right-circle" title="6. Understand the Keywords">
    Each test step can use different **keywords** that represent actions.  
    For example:
    - `navigate` → Opens a URL in the browser  
    - `click`, `verifyText`, `type`, and more…

    These keywords help automate all possible web scenarios.

  <a href="/keywords" class="button-link">
    Learn about all keywords
  </a>

    <img src="/imgs/quickstart/Nexgen.png" alt="Keyword list for web testing" />
  </Accordion>
</AccordionGroup>

---

### Step 2: Attach Test Data

<AccordionGroup>
  <Accordion icon="table" title="1. Create Test Data">
    Go to the **Test Data** section under your selected test case.  
    <img src="/imgs/quickstart/Nexgen.png" alt="Create test data table" />
  </Accordion>

  <Accordion icon="edit" title="2. Add values to the test data table">
    Fill in your **input values**, **expected outputs**, or any **dynamic data** needed for your case.  
    <img src="/imgs/quickstart/Nexgen.png" alt="Add data values to test case" />
  </Accordion>

  <Accordion icon="save" title="3. Save your test data">
    Once done, click **Save**.  
    Your test case is now ready to execute.  
    <img src="/imgs/quickstart/Nexgen.png" alt="Save test data" />
  </Accordion>
</AccordionGroup>

---

### Step 3: Run Your Test Case

<Accordion icon="rocket" title="Run and Validate">
  1. Go to the **Run Test Case** section.  
  2. Choose your test case and click **Run**.  
  3. Observe logs and results in real-time.  
  4. Once the test passes, you'll see a success notification.

  <Tip>Congratulations! You've just executed your first automated web test case.</Tip>
  <img src="/imgs/quickstart/Nexgen.png" alt="Running and validating test case" />
</Accordion>
