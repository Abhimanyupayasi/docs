# Executing Visual Testing in NextGen QE

Follow these steps to perform **Visual Testing (VT)** in **NextGen QE** using the **XBrowsIQ** tab.  
This process helps validate UI consistency across browsers and detects layout or visual discrepancies automatically.

---

## Steps to Execute Visual Testing

### 1. Open the XBrowsIQ Tab
Go to **NexGen QE → XBrowsIQ Tab** to start configuring your visual tests.

<img src="/web-testing/imgs/vt_1.png" alt="Navigating to XBrowsIQ tab in NexGen QE for visual testing setup" />

---

### 2. Add a Test Suite
Click on **Add Test Suite** to include the test suite you want to perform visual comparison on.

<img src="/web-testing/imgs/vt_2.png" alt="Navigating to XBrowsIQ tab in NexGen QE for visual testing setup" />

---

### 3. Create a Baseline
Select the desired test suite and click **Create Baseline**.  
This captures reference screenshots used for future comparisons.

---

### 4. Configure Test Preferences
A configuration panel will appear — update it with your preferred **browser settings, viewport sizes**, and **comparison options**.

<img src="/web-testing/imgs/vt_3.png" alt="Navigating to XBrowsIQ tab in NexGen QE for visual testing setup" />


---

### 5. Execute Visual Testing
After configuration, click the **VT (Visual Test)** button to execute the visual testing process.



---

### 6. Review Visual Testing Report
Once execution completes, you can view the **Visual Testing Report**.  
It highlights all detected visual differences between your **baseline** and **latest execution**.



---

> ✅ **Result:** You’ll be able to clearly identify layout shifts, styling errors, or broken elements through automated visual comparison.
